Lua 中pairs使用过程中碰到的一个很奇葩的问题

· 2019-Dec-13 · lua bug record

在使用pairs遍历一个普通的table时，我发现无论怎样，都会在循环进行一次后退出循环

问题代码

local t = {
    111 = {
        
    },
    222 = {

    }
}
for index, val in pairs(t) do
    index = tostring(index)
    -- do something
end

问题

这个问题很容易被忽略，在经过一段不短时间的纠结后，我终于发现了问题所在。问题出在在循环中对index变量的处理中，即index = tostring(index)。

原因

这个问题的产生是因为使用pairs遍历table时，实际上使用next来遍历table，而next会使用index作为参数，因此循环中对于index的修改可能导致next无法找到值为index的key，返回nil，终止循环。
